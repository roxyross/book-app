API Reference

Overview​

This document provides a comprehensive reference for all API endpoints in the AI Book Platform, including both frontend Next.js API routes and backend FastAPI endpoints.

Backend API (FastAPI) - http://localhost:8000​

Authentication Endpoints​

POST /auth/login​

Authenticate a user and receive a JWT token.

Request:

{
  "email": "user@example.com",
  "password": "user_password"
}

Response:

{
  "access_token": "jwt_token_here",
  "token_type": "bearer",
  "user": {
    "id": "user_id",
    "email": "user@example.com",
    "role": "user"
  }
}

POST /auth/logout​

Logout the current user.

Response:

{
  "message": "Successfully logged out"
}

Chat Endpoints​

POST /chat​

Send a query to the RAG-powered chatbot.

Headers:

Authorization: Bearer <jwt_token>
Content-Type: application/json

Request:

{
  "query": "Your question about AI concepts",
  "user_id": "optional_user_id"
}

Response:

{
  "response": "Generated response based on curriculum content",
  "citations": [
    {
      "title": "Source title",
      "source": "source_file.mdx",
      "relevance_score": 0.95
    }
  ],
  "user_id": "user_id"
}

Content Management Endpoints​

POST /load-content​

Load and index all MDX content into the vector database.

Response:

{
  "message": "Content loaded and indexed successfully"
}

Health Check Endpoints​

GET /​

Get API information and status.

Response:

{
  "message": "AI Book Platform RAG API"
}

GET /health​

Check the health status of the API.

Response:

{
  "status": "healthy"
}

Frontend API (Next.js) - http://localhost:3000​

Better Auth Endpoints​

The frontend uses Better Auth which provides the following endpoints:

GET/POST /api/auth/signin​

Handle user sign-in (email/password, OAuth, etc.)

GET/POST /api/auth/signout​

Handle user sign-out

GET/POST /api/auth/signup​

Handle user registration

GET /api/auth/session​

Get current user session information

Frontend Client-Side API Usage​

Chat Component API Integration​

The chatbot component makes requests to the backend API:

const response = await fetch('http://localhost:8000/chat', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    query: input,
    user_id: session.user.id,
  }),
});

Authentication Integration​

The frontend uses Better Auth client-side hooks:

import { useSession } from '@/hooks/useSession';

const { data: session, isPending } = useSession();

Error Handling​

Backend Error Responses​

All backend errors follow this structure:

{
  "detail": "Error message"
}

HTTP Status Codes​

  • 200 - Success
  • 201 - Created
  • 400 - Bad Request
  • 401 - Unauthorized
  • 404 - Not Found
  • 500 - Internal Server Error

Rate Limiting​

The backend API includes rate limiting to prevent abuse:

  • Unauthenticated requests: 100/hour
  • Authenticated requests: 1000/hour
  • Chat endpoint: 50/hour per user (to control costs)

Request/Response Formats​

Common Request Headers​

  • Content-Type: application/json
  • Authorization: Bearer <token> (for protected endpoints)

Common Response Headers​

  • Content-Type: application/json
  • X-RateLimit-Limit: Request limit
  • X-RateLimit-Remaining: Remaining requests
  • X-RateLimit-Reset: Time when limit resets

Authentication Requirements​

Public Endpoints​

  • GET /
  • GET /health

Authenticated Endpoints​

  • POST /chat
  • POST /load-content (admin only)

Content Structure​

MDX Content Format​

The system expects MDX files with frontmatter:

---
title: "Content Title"
description: "Content description"
---

# Content Title

## Topic 1: Topic Title

Content goes here...

Content Processing​

  1. Frontmatter is extracted for metadata
  2. Markdown content is converted to plain text
  3. Text is chunked into ~1000 character segments
  4. Each chunk is embedded and stored with metadata
