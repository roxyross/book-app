Deployment

Overview​

This guide provides instructions for deploying the AI Book Platform to production environments. The platform consists of three main components that can be deployed separately or together depending on your infrastructure preferences.

Architecture Overview​

The AI Book Platform consists of:

  1. Frontend: Next.js application (runs on port 3000 by default)
  2. Backend: FastAPI RAG system (runs on port 8000 by default)
  3. Vector Database: Qdrant (runs on port 6333 by default)
  4. Authentication Database: SQLite/PostgreSQL for Better Auth

Deployment Options​

Option 1: All-in-One Deployment​

Deploy all components on a single server or container platform.

Requirements:

  • Server with at least 4GB RAM
  • Node.js, Python, and Docker installed
  • SSL certificate for HTTPS

Option 2: Microservices Deployment​

Deploy each component separately for better scalability.

Components:

  • Frontend on Vercel/Netlify
  • Backend on AWS/GCP/Azure
  • Qdrant on Qdrant Cloud or self-hosted
  • Database on managed service

Option 3: Container Deployment​

Use Docker containers for all components.

Production Configuration​

Environment Variables​

Frontend (.env.production)​

# Production URL
NEXT_PUBLIC_APP_URL=https://your-domain.com
NEXT_PUBLIC_BETTER_AUTH_URL=https://your-domain.com

# Better Auth
BETTER_AUTH_SECRET=your-production-secret-key
DATABASE_URL=postgresql://user:password@db-host:5432/ai_book_platform

# Analytics (optional)
NEXT_PUBLIC_ANALYTICS_ID=your-analytics-id

Backend (.env.production)​

# API Keys
COHERE_API_KEY=your-production-cohere-api-key
JWT_SECRET=your-production-jwt-secret

# Vector Database
QDRANT_URL=your-qdrant-url.com
QDRANT_API_KEY=your-qdrant-api-key
QDRANT_PORT=6333

# Production settings
ENVIRONMENT=production
LOG_LEVEL=info

Deployment Steps​

1. Prepare the Frontend​

cd book-app
npm run build

2. Prepare the Backend​

cd backend
pip install -r requirements.txt

3. Set up Production Database​

For production, consider using PostgreSQL instead of SQLite:

# Install PostgreSQL and create database
createdb ai_book_platform

Update the Better Auth configuration to use PostgreSQL.

4. Deploy Frontend​

To Vercel (Recommended)​

npm install -g vercel
vercel --prod

To Other Platforms​

  • Netlify: Use the build command npm run build and publish directory out
  • AWS S3: Build and deploy the static files
  • Traditional Server: Serve the built files with nginx or Apache

5. Deploy Backend​

Using PM2 (Node.js server)​

npm install -g pm2
pm2 start "cd backend && python main.py" --name "ai-book-backend"

Using Docker​

Create a Dockerfile for the backend:

FROM python:3.11-slim

WORKDIR /app
COPY backend/requirements.txt .
RUN pip install -r requirements.txt

COPY backend/ .

EXPOSE 8000
CMD ["python", "main.py"]

Build and run:

docker build -t ai-book-backend .
docker run -d -p 8000:8000 --env-file .env ai-book-backend

6. Set up Qdrant​

Qdrant Cloud (Recommended for Production)​

  1. Sign up at qdrant.tech
  2. Create a new cluster
  3. Get your URL and API key
  4. Update your backend configuration

Self-hosted Qdrant​

docker run -d --name qdrant-container -p 6333:6333 -v $(pwd)/qdrant_storage:/qdrant/storage qdrant/qdrant

SSL/HTTPS Configuration​

Using Let's Encrypt​

sudo apt-get install certbot python3-certbot-nginx
sudo certbot --nginx -d your-domain.com

Nginx Configuration​

server {
    listen 443 ssl http2;
    server_name your-domain.com;

    ssl_certificate /path/to/certificate.crt;
    ssl_certificate_key /path/to/private.key;

    location / {
        proxy_pass http://localhost:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    location /api/ {
        proxy_pass http://localhost:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

Performance Optimization​

Frontend Optimizations​

  1. Image Optimization: Use Next.js Image component
  2. Code Splitting: Leverage Next.js automatic splitting
  3. Caching: Implement proper HTTP caching headers
  4. CDN: Use a CDN for static assets

Backend Optimizations​

  1. Database Indexing: Properly index Qdrant collections
  2. Caching: Implement Redis for frequently accessed data
  3. Load Balancing: Use multiple backend instances
  4. Monitoring: Set up health checks and monitoring

Vector Database Optimizations​

  1. Collection Configuration: Optimize for your query patterns
  2. Hardware: Use appropriate instance types for your data size
  3. Replication: Set up replicas for high availability
  4. Monitoring: Monitor query performance and resource usage

Monitoring and Maintenance​

Health Checks​

  • Frontend: /health endpoint
  • Backend: /health endpoint
  • Qdrant: / endpoint

Logging​

  • Set up structured logging
  • Centralize logs using services like ELK stack
  • Monitor error rates and performance metrics

Backup Strategy​

  1. Database Backup: Regular PostgreSQL backups
  2. Content Backup: Version control for MDX files
  3. Vector Database: Qdrant storage backup
  4. Configuration: Version control for environment files

Scaling Considerations​

Horizontal Scaling​

  • Frontend: CDN and static hosting
  • Backend: Multiple instances behind load balancer
  • Database: Read replicas and connection pooling
  • Vector DB: Qdrant cluster mode

Vertical Scaling​

  • Increase server resources (CPU/RAM)
  • Optimize database queries
  • Implement caching layers
  • Use content delivery networks

Security Best Practices​

  1. HTTPS: Always use HTTPS in production
  2. Secrets: Use environment variables for secrets
  3. Authentication: Implement proper session management
  4. Input Validation: Validate all user inputs
  5. Rate Limiting: Implement API rate limiting
  6. Security Headers: Add security headers to responses
