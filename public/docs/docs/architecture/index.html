Architecture

Overview​

The AI Book Platform is a comprehensive learning system that combines educational content with AI-powered assistance. The architecture follows modern web development practices with a clear separation of concerns.

System Components​

Frontend (Next.js App Router)​

  • Technology: Next.js 15 with App Router
  • Components: Server Components by default, Client Components only when needed
  • Styling: Tailwind CSS + shadcn/ui
  • Authentication: Better Auth integration

Backend (Python/FastAPI)​

  • Technology: FastAPI for the RAG system
  • Responsibilities:
    • Load MDX content from the frontend
    • Chunk content by chapter/topic
    • Embed using Cohere
    • Store/search vectors in Qdrant
    • Enforce strict RAG (never answer outside retrieved context)

RAG Pipeline​

  1. Content Loading: Load MDX files from book-app/src/content/chapters
  2. Content Processing: Parse MDX to extract text content
  3. Chunking: Split content into manageable chunks (~1000 characters)
  4. Embedding: Generate embeddings using Cohere's embed-multilingual-v3.0 model
  5. Storage: Store vectors in Qdrant vector database
  6. Retrieval: Search relevant content based on user queries
  7. Generation: Generate responses using Cohere's language model with retrieved context

Authentication System​

  • Technology: Better Auth
  • Features: Login & logout flows, role-based access control
  • Access: Chatbot access is auth-gated, reading content remains public

Data Flow​

User Query → Cohere Embedding → Qdrant Search → Retrieved Context → Cohere Generation → Response

Security & Access Control​

  • Public Content: Book chapters and topics are accessible without authentication
  • Protected Features: AI chatbot requires authentication
  • Role-Based Access: Different user roles have different permissions
  • Secure API: Backend API endpoints are protected and validated
