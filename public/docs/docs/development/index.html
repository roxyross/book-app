Development

Overview​

This guide covers the development workflow for the AI Book Platform. Learn how to extend the curriculum, enhance the RAG system, and customize the platform for your needs.

Project Structure​

ai-book-platform/
├── book-app/                 # Next.js frontend application
│   ├── src/
│   │   ├── app/             # App Router pages
│   │   ├── components/      # React components
│   │   ├── content/         # Curriculum MDX files
│   │   └── lib/             # Utility functions
│   └── ...
├── backend/                  # FastAPI RAG system
│   ├── main.py              # Main application
│   └── ...
└── docs/                     # Docusaurus documentation

Adding New Curriculum Content​

1. Create MDX Files​

Add new MDX files to book-app/src/content/chapters/ with the following structure:

---
title: "Your Chapter Title"
description: "Brief description of the content"
---

# Your Chapter Title

## Topic 1: Topic Title

Content for the first topic...

## Topic 2: Topic Title

Content for the second topic...

2. Update Navigation​

The curriculum navigation automatically includes all MDX files in the chapters directory. No additional configuration is needed.

3. Re-index Content​

After adding new content, restart the backend server to re-index the content into the vector database, or call the /load-content endpoint.

Extending the RAG System​

Adding New Content Sources​

To add content from other sources:

  1. Update the load_and_index_mdx_content() function in backend/main.py
  2. Create new functions to load content from different formats
  3. Ensure the chunking and embedding process works with the new content

Improving Search Relevance​

The search functionality can be enhanced by:

  1. Adjusting the chunk size in the chunk_text() function
  2. Experimenting with different embedding models
  3. Implementing hybrid search combining vector and keyword search

Response Generation​

The response generation can be improved by:

  1. Updating the prompt engineering in the /chat endpoint
  2. Implementing response validation to ensure accuracy
  3. Adding response formatting for better readability

API Endpoints​

Backend API (http://localhost:8000)​

  • GET / - Health check and API info
  • POST /auth/login - User authentication
  • POST /auth/logout - User logout
  • POST /chat - Chat with RAG-powered assistant
  • POST /load-content - Load and index curriculum content
  • GET /health - Health check

Frontend API (http://localhost:3000)​

  • Better Auth endpoints at /api/auth/*
  • Next.js API routes in book-app/src/app/api/

Environment Configuration​

Frontend Environment Variables​

# Better Auth
NEXT_PUBLIC_BETTER_AUTH_URL=http://localhost:3000
BETTER_AUTH_SECRET=your-secret-key

# Database
DATABASE_URL=file:./db.sqlite

# Application
NEXT_PUBLIC_APP_URL=http://localhost:3000

Backend Environment Variables​

# API Keys
COHERE_API_KEY=your-cohere-api-key
JWT_SECRET=your-jwt-secret

# Vector Database
QDRANT_URL=localhost
QDRANT_PORT=6333
QDRANT_API_KEY=your-qdrant-key-if-using-cloud

Testing​

Frontend Testing​

  1. Unit tests for React components
  2. Integration tests for API interactions
  3. End-to-end tests for user flows

Backend Testing​

  1. Unit tests for utility functions
  2. Integration tests for API endpoints
  3. Load testing for RAG performance

Performance Optimization​

Frontend Optimizations​

  • Use server components by default
  • Implement proper caching strategies
  • Optimize images and assets
  • Minimize JavaScript bundle size

Backend Optimizations​

  • Optimize vector database queries
  • Implement proper indexing strategies
  • Use async/await for I/O operations
  • Cache frequently accessed content

Security Considerations​

  • Validate all user inputs
  • Sanitize content before processing
  • Implement proper authentication
  • Use HTTPS in production
  • Regular dependency updates

Deployment​

Frontend Deployment​

The Next.js application can be deployed to:

  • Vercel (recommended)
  • Netlify
  • AWS, GCP, Azure
  • Any Node.js hosting platform

Backend Deployment​

The FastAPI application can be deployed to:

  • AWS, GCP, Azure
  • Docker containers
  • Serverless platforms (with modifications)
  • Traditional VPS

Database Considerations​

For production:

  • Use PostgreSQL instead of SQLite
  • Implement proper backup strategies
  • Monitor database performance
  • Set up read replicas for high traffic
