Installation

Prerequisites​

Before installing the AI Book Platform, ensure you have the following installed:

  • Node.js: Version 18.0 or higher
  • Python: Version 3.9 or higher
  • npm: Version 8.0 or higher (usually comes with Node.js)
  • Git: For version control
  • Qdrant: Vector database (can run locally or use cloud version)

Quick Start​

1. Clone the Repository​

git clone https://github.com/ai-book-platform/ai-book-platform.git
cd ai-book-platform

2. Set up the Frontend (Next.js)​

cd book-app
npm install

Create a .env.local file in the book-app directory:

NEXT_PUBLIC_BETTER_AUTH_URL=http://localhost:3000
BETTER_AUTH_SECRET=your-secret-key-change-in-production
DATABASE_URL=file:./db.sqlite

3. Set up the Backend (FastAPI)​

cd ../backend
pip install -r requirements.txt

Create a .env file in the backend directory:

COHERE_API_KEY=your-cohere-api-key
QDRANT_URL=localhost
QDRANT_PORT=6333
QDRANT_API_KEY=your-qdrant-api-key-if-using-cloud
JWT_SECRET=your-jwt-secret-change-in-production

4. Set up Qdrant​

Option A: Docker (Recommended for Development)​

docker run -d --name qdrant-container -p 6333:6333 -p 6334:6334 qdrant/qdrant

Option B: Qdrant Cloud​

Sign up at qdrant.tech and get your API key and URL.

Environment Variables​

Frontend (book-app/.env.local)​

# Better Auth configuration
NEXT_PUBLIC_BETTER_AUTH_URL=http://localhost:3000
BETTER_AUTH_SECRET=your-very-secure-secret-key-change-in-production

# Database configuration
DATABASE_URL=file:./db.sqlite

# Other configurations
NEXT_PUBLIC_APP_URL=http://localhost:3000

Backend (backend/.env)​

# API Keys
COHERE_API_KEY=your-cohere-api-key-here
JWT_SECRET=your-very-secure-jwt-secret-change-in-production

# Qdrant Configuration
QDRANT_URL=localhost
QDRANT_PORT=6333
# QDRANT_API_KEY=your-qdrant-api-key-if-using-cloud

# Development settings
ENVIRONMENT=development

Running the Application​

1. Start the Backend Server​

cd backend
python main.py

The backend will start on http://localhost:8000 and automatically load the curriculum content.

2. Start the Frontend Server​

In a new terminal:

cd book-app
npm run dev

The frontend will start on http://localhost:3000.

3. Access the Application​

Open your browser and navigate to http://localhost:3000.

Database Setup​

The application uses SQLite for simplicity in development. For production, consider PostgreSQL or another production-ready database.

The Better Auth database will be automatically created when the application starts for the first time.

Content Management​

All curriculum content is stored in MDX files in:

book-app/src/content/chapters/

After adding or modifying content, restart the backend to re-index the content into the vector database.

Troubleshooting​

Common Issues​

  1. Backend can't connect to Qdrant

    • Ensure Qdrant is running on the specified URL and port
    • Check that the QDRANT_URL and QDRANT_PORT match your Qdrant instance

  2. Cohere API errors

    • Verify your COHERE_API_KEY is valid
    • Check that your Cohere account has sufficient credits

  3. Content not appearing in chat

    • Ensure the backend has successfully loaded content on startup
    • Check that the MDX files are in the correct location
    • Verify that the content has been indexed by calling /load-content endpoint

Development Notes​

  • The application automatically loads content on startup
  • Any changes to MDX files require restarting the backend to re-index
  • The chatbot will only respond to queries related to the curriculum content
