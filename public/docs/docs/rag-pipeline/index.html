RAG Pipeline

Overview​

The Retrieval-Augmented Generation (RAG) pipeline is the core of the AI Book Platform's intelligent assistance system. It enables the chatbot to answer questions based on the curriculum content while enforcing strict adherence to the provided context.

Pipeline Components​

1. Content Loading​

  • Source: MDX files from book-app/src/content/chapters/
  • Process: Reads all .mdx files and extracts text content
  • Metadata: Preserves source file information and chunk indices

2. Content Processing​

  • Frontmatter Parsing: Extracts title and description from MDX frontmatter
  • Markdown Conversion: Converts MDX content to HTML, then to plain text
  • Text Cleaning: Normalizes whitespace and removes excessive newlines

3. Chunking Strategy​

  • Method: Paragraph-based chunking
  • Size: Approximately 1000 characters per chunk
  • Logic: Preserves paragraph boundaries to maintain context coherence

4. Embedding Generation​

  • Model: Cohere's embed-multilingual-v3.0
  • Dimensions: 4096-dimensional vectors
  • Storage: Vectors stored in Qdrant with content metadata

5. Vector Storage​

  • Database: Qdrant vector database
  • Collection: ai_book_content
  • Schema: Content, source file, chunk index, and title
  • Distance: Cosine similarity for similarity search

6. Retrieval Process​

  • Query Embedding: User queries are embedded using the same model
  • Search: Vector similarity search in Qdrant
  • Results: Top-k most relevant content chunks returned
  • Metadata: Source attribution preserved for citations

7. Response Generation​

  • Context Assembly: Retrieved chunks are combined into context
  • Prompt Engineering: Context and query are formatted for the LLM
  • Generation: Cohere's command-xlarge-nightly model generates response
  • Citation: Source files are provided for transparency

RAG Rules​

Strict Context Adherence​

  • Rule: Never answer outside retrieved context
  • Implementation: If no relevant context is found, the system refuses to answer
  • Fallback: Directs users to rephrase or ask about covered topics

Citation Requirement​

  • Rule: Always cite chapter + topic when answering
  • Implementation: Source metadata is preserved and returned with responses
  • Format: Citations include source file and relevance score

Quality Assurance​

  • Relevance Threshold: Only highly relevant chunks are included in context
  • Context Validation: Ensures retrieved content is directly related to the query
  • Response Verification: Generated responses are checked against source content

API Endpoints​

/load-content (POST)​

  • Purpose: Load and index all MDX content into vector database
  • Trigger: Called on application startup and manually when needed
  • Process: Full re-indexing of all curriculum content

/chat (POST)​

  • Purpose: Process user queries using the RAG pipeline
  • Input: User query and optional user ID
  • Output: Generated response with citations
  • Process: Embed query → Search vectors → Generate response

Performance Considerations​

  • Indexing Time: Initial content loading may take several minutes
  • Query Latency: Vector search and response generation under 2 seconds
  • Memory Usage: Optimized for efficient vector storage and retrieval
  • Scalability: Qdrant supports horizontal scaling for larger datasets
